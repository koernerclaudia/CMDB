"cMDb" - Movie Database API

Building a server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints - Movie database

Your request Endpoint / URL HTTP Method Request body data format Response body data format
Get a list of all movies in the database /movies GET None A JSON object holding all data of every movie.
Get a specific movie title /movies/[title]

Exampe:
/movies/Black%20Swan
 GET None A JSON object holding data about the movie title containing the title, name of director, birth year of director, genre and genre description.

"title": "Black Swan",
"director": {
   "name": "Darren Aronofsky",
   "birth": "1969"
},
"genre": {
   "name": "Thriller",
   "description": "Gives you the chills." }
Get a specific genre and its description /movies/genre/[name]

Exampe:
/movies/genre/Thriller
 GET None A JSON object holding data about the genre title and its description.

{
"name": "Thriller",
"description": "Gives you the chills.
}
Get information on the director /movies/directors/[name]

Exampe:
/movies/directors/Steven%20Spielberg
 GET None A JSON object holding data about the director's name and his year of birth.

{
"name": "Steven Spielberg",
"birth": "1946"
}

Endpoints - User Details

User request Endpoint / URL HTTP Method Request body data format Response body data format
Create a new user /users POST A JSON object holding data about the user's name and a placeholder for their favourite movies, structured like this

{
"name": "Marie",
"favMovies": "[ ]"
}
 A JSON object holding data about the user that has been added, a placeholder for their favourite movie(s), as well as an assigned ID.

{
"name": "Marie",
"favMovies": "[ ]",
"id": "8a9d8fcf-d43a-4ea2-bb90-6fe3efcc3050"
}
Change/ Update the name of a user /users/[id]

Exampe:
/users/2
 PUT A JSON object with the updated name of the user, structured like this

{
"name": "Thomas"
}
 A JSON file showing the user information with the newly updated name, including the user ID and their favourite movies.
{
"name": "Thomas",
"favMovies": "Interstellar",
"id": "2"
}
Let the user add a movie to their favourites /users/[ID]/[movieTitle]

Exampe:
/users/2/Black%20Swan
 POST A JSON file containing the object key for favourite movies and the value left empty.

{
"favMovies": [ ],
}
 A text file returning a statement to confirm the change has been made, such as:

Black Swan has been added to the user 1's array.
Let the user delete a movie from their favourites /users/[ID]/[movieTitle]

Exampe:

 DELETE A JSON file containing the object key for favourite movies and the value left empty.

}
